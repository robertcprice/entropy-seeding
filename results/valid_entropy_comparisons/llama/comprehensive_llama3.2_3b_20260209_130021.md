## Appendix: Metrics Glossary & Interpretation Guide

### Metrics Used in This Report

| Metric | Full Name | Range | Interpretation |
|--------|-----------|-------|----------------|
| **shannon_char** | Character-level Shannon entropy | 0 -- 4.7 bits | Measures the information density of individual characters in the output. Higher values indicate more varied character usage (more uniform distribution across the alphabet, punctuation, etc.). Values near 4.7 approach the theoretical maximum for printable ASCII. Values below 4.0 suggest repetitive character patterns. |
| **shannon_word** | Word-level Shannon entropy | 0 -- 12 bits | Measures the information density of the word distribution. Higher values indicate a broader active vocabulary in the output. This metric is sensitive to output length: longer outputs tend to have higher word entropy because more unique words appear. Typical LLM outputs range from 6.0 to 8.5 bits. |
| **word_diversity** | Type-Token Ratio (TTR) | 0 -- 1 | Calculated as unique_words / total_words. Higher values indicate more lexically diverse output with less word repetition. Unlike shannon_word, TTR is inversely correlated with length (longer texts naturally reuse more words), so comparisons are most meaningful when length_words values are similar across conditions. Values above 0.7 indicate high diversity; below 0.5 suggests significant repetition. |
| **length_words** | Output length in words | 0 -- unbounded | A control metric measuring the total word count of the generated output. This is not a quality metric per se, but is essential for interpreting the other metrics. Differences in length between entropy conditions (PRNG, TRNG, QRNG) can confound shannon_word and word_diversity comparisons. When length varies substantially across conditions for a given prompt, interpret diversity metrics with caution. |

### Entropy Source Conditions

| Condition | Description |
|-----------|-------------|
| **PRNG** | Pseudorandom number generator (deterministic, algorithmic randomness). Serves as the baseline/control condition. |
| **TRNG** | True random number generator (hardware-based thermal/atmospheric noise). Non-deterministic physical randomness. |
| **QRNG** | Quantum random number generator (quantum vacuum fluctuation-based). Theoretically the highest-quality entropy source. |

### Reading the Results Tables

Each table shows the **mean value** across 5 samples per condition for a given prompt. When comparing across entropy conditions (PRNG vs TRNG vs QRNG):

- **Small differences** (e.g., shannon_char varying by < 0.05) are likely within normal generation variance and should not be interpreted as meaningful effects of the entropy source.
- **Consistent directional trends** across multiple prompts are more informative than any single prompt comparison.
- **length_words** should be checked first: if one condition produces substantially longer or shorter outputs, the other metrics may shift as a consequence of length rather than entropy quality.

### Model-Specific Note: Llama 3.2 3B

Meta's Llama 3.2 3B uses a dense transformer architecture with **Grouped-Query Attention (GQA)**, which shares key-value heads across multiple query heads to reduce memory bandwidth requirements while preserving attention quality. At 3 billion parameters, this is one of the smaller models in the Llama 3.2 family, and its outputs show characteristic patterns of smaller models: notably lower word_diversity values (frequently 0.5--0.7, with some prompts dipping below 0.5 on technical topics like neural networks) and more formulaic response structures. The relatively tight clustering of metrics across PRNG, TRNG, and QRNG conditions suggests that at this parameter scale, the model's learned priors dominate over entropy source variation -- the model's limited capacity constrains the output distribution more tightly than the randomness quality can perturb it. This is consistent with the broader finding that larger models show greater sensitivity to entropy source differences.
